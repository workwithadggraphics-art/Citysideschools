🔒 IP Restriction — How It Works

To prevent students from logging in from home, all student computers must be connected to the same local network (LAN / Wi-Fi) as the admin machine. The backend server runs on your machine and only accepts requests from IP addresses within your school's subnet.

Students open the portal by navigating to your machine's local IP in their browser, e.g. http://192.168.1.100:3000 — not a public URL. Anyone outside the school network cannot reach it.

To find your local IP: open Command Prompt → type ipconfig (Windows) or open Terminal → ifconfig (Linux/Mac). Look for IPv4 Address under your active adapter.

📦 Deployment Summary

Option A — Netlify + GitHub (no IP restriction, public URL):
Push student.html and admin.html to a GitHub repo. Connect to Netlify. Both files deploy automatically. Data is stored in localStorage — each browser maintains its own data.

Option B — Local School Server (with IP restriction, recommended for exams):
Run a Node.js/Express server on the admin PC. Place both HTML files in the /public folder. Students access via the server's local IP. The backend validates IP on every login request and blocks external connections. Use Firebase Firestore for shared data across all machines on the network.

When you're ready to build the Node.js backend, Claude can generate it for you — it will include Firebase integration, IP middleware, and a REST API for login/results.
